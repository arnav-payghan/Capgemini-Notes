# 02-02-2026: ABAP Programming Techniques.

### Performance Tools

- **Extended Program Check (SLIN)**: Checks the syntax of code and gives output - error, warning and checks.
- **Code Inspector (SCI)**: Checks on Performace, Security and Syntax.
- **Runtime Analysis (SE30)**: Overall execution time in microseconds.
- **SQL Trace (ST05)**: All indexes being hit (execution time). How much time did the SQL query take to execute in microsecond. (FM as well). Has different types of traces.
- **ABAP Test Cockpit (ATC)**: All naming conventions based on project standards. ATC Check, shows all issues. There should be no ATC errors. There should be no Priority 1 errors.
- **ABAP DUMP (ST22)**: Shows all the dumps, for us to analyze these dumps. Based on client agreement dump lifeline is set.

> ATC will be configured by the Team Lead.

> Trace - Any activity done, will be logged step by step with timestamps.

> BAPI to change standard table data.

```
FORM <f_cat_pop> USING <f_field> <f_tab> <f_label> <f_pos>.
    <gs_fcat>-fieldname = <f_field>.    " <gs_fcat> is work area.
    <gs_fcat>-f_tab = <f_tab>.
    <gs_fcat>-label = <f_tab>.
    <gs_fcat>-col-pos = <f_pos>.

    APPEND <gs_fcat> TO <gt_fcat>.      " <gt_fcat> is internal table
ENDFORM.
```
